## 🔍 验证方法

### 方法 1：查看浏览器控制台

打开开发者工具（F12），查看 Console 标签页，应该看到：

```
[CapacityPanel Debug] historicalData length: 50000
[CapacityPanel Debug] historicalData first item: {time: 1741348060242, value: 1023.45}
[CapacityPanel Debug] historicalData last item: {time: 1741391260242, value: 1045.67}
```

### 方法 2：检查 X 轴时间范围

将鼠标悬停在图表上，tooltip 应该显示：

```
2026/3/7 14:23:45

```

**不是**：
- ❌ `Invalid Date`
- ❌ `N/A`

在控制台执行：

```javascript

```

或直接查看：

- **X 轴最小值**：12 小时前的时间戳
- **X 轴最大值**：当前时间戳
- **Y 轴范围**：约 950 ~ 1100

### 方法 4：模拟新数据追加

等待 30 秒（或手动刷新），图表应该自动追加新数据点，历史数据保持 50,000+ 条。

## 🚨 常见问题排查

### 问题 1：一直显示"数据加载中"

**原因**：
- `queryResult.isLoading` 始终为 true
- Vue Query 没有正确初始化

**解决**：
- 检查 `queryResult.isLoading` 是否为 `false`
- 检查 `queryResult.data` 是否有值

### 问题 2：图表显示"Invalid Date"

**原因**：
- 数据格式不是 `[timestamp, value]`
- ECharts 无法解析时间轴

**解决**：
- 确保数据转换为 `[item.time, item.value]` 格式

### 问题 3：图表显示"NaN"或"N/A"

**原因**：
- Tooltip formatter 接收到 undefined
- 数据中有 undefined 值

**解决**：
- 添加数据过滤：`if (!param || param.value === undefined) return '';`

### 问题 4：历史数据只有几百条

**原因**：
- `initializeHistoricalData()` 没有执行
- watch 没有触发

**解决**：
- 检查控制台是否有 `历史数据初始化` 日志
- 检查 `historicalData.length` 是否为 50000
