### 2.1 消除魔法数字（常量抽离）

#### 重构前
```typescript
// helpers.ts
export function calculateScale(
  containerWidth: number,
  containerHeight: number,
  designWidth: number = 1920,      // ❌ 魔法数字硬编码
  designHeight: number = 1080      // ❌ 魔法数字硬编码

// ScaleBox.vue
const props = withDefaults(defineProps<ScaleBoxProps>(), {
  width: 1920,                      // ❌ 魔法数字硬编码
  height: 1080,                     // ❌ 魔法数字硬编码

```

#### 重构后
```typescript
// ✅ 新建 constants.ts
export const DEFAULT_DESIGN_WIDTH = 1920;
export const DEFAULT_DESIGN_HEIGHT = 1080;
export const SCALE_BOX_DEFAULT_PROPS = {
  width: DEFAULT_DESIGN_WIDTH,
  height: DEFAULT_DESIGN_HEIGHT,
} as const;

// ✅ helpers.ts 引用常量
import { DEFAULT_DESIGN_WIDTH, DEFAULT_DESIGN_HEIGHT } from './constants';
export function calculateScale(
  containerWidth: number,
  containerHeight: number,
  designWidth: number = DEFAULT_DESIGN_WIDTH,  // ✅ 统一引用
  designHeight: number = DEFAULT_DESIGN_HEIGHT // ✅ 统一引用

// ✅ ScaleBox.vue 引用常量
import { DEFAULT_DESIGN_WIDTH, DEFAULT_DESIGN_HEIGHT } from './constants';

  width: DEFAULT_DESIGN_WIDTH,  // ✅ 统一引用
  height: DEFAULT_DESIGN_HEIGHT, // ✅ 统一引用
```

#### 优化价值
- ✅ **单一数据源**：修改设计尺寸仅需改 `constants.ts`
- ✅ **类型安全**：`as const` 确保常量不可变
- ✅ **可维护性**：大写蛇形命名法提升代码可读性

### 2.2 优化首屏动画抖动

#### 重构前
```typescript
// ❌ 问题：所有 scale 变化都带动画，首屏计算时会产生"缩放抖动"
const scaleStyle = computed(() => ({
  transform: `scale(${scale.value})`,
  transition: 'transform 0.3s ease-out',  // ❌ 首屏也有动画
}));
```

#### 重构后
```typescript
// ✅ 首屏不加 transition，仅在 resize 时启用
let isFirstMount = true;

const scaleStyle = computed(() => {
  // 📌 首次挂载不加 transition
  const hasTransition = !isFirstMount && containerRef.value !== null;
  
  return {
    transform: `scale(${scale.value})`,
    transition: hasTransition ? 'transform 0.3s ease-out' : 'none',
  };
});

// 🏁 挂载完成后标记（使用 requestAnimationFrame 避免首屏抖动）
onMounted(() => {
  requestAnimationFrame(() => {
    isFirstMount = false;
  });
});

// 🧹 卸载时重置标记（防止组件复用问题）
onBeforeUnmount(() => {
  isFirstMount = true;
});
```

#### 优化价值
- ✅ **消除首屏抖动**：首屏渲染无动画，避免视觉"跳动"
- ✅ **动态启用 transition**：resize 时仍有平滑过渡（用户体验优化）
- ✅ **requestAnimationFrame**：确保在浏览器下一帧渲染前标记（性能优化）

### 2.3 CSS 防溢出强化

#### 重构前
```css
/* ❌ 问题：样式不够严格，极端缩放比例下可能出现滚动条 */
.scalebox-wrapper {
  width: 100vw;
  height: 100vh;
  overflow: hidden;  /* ❌ 仅单层 overflow */
  /* ... */
}
```

#### 重构后
```css
/* ✅ 防溢出强化（三保险策略） */
.scalebox-wrapper {
  width: 100vw;
  height: 100vh;
  
  /* 🛡️ 三重 overflow 保护 */
  overflow: hidden;
  overflow-x: hidden;
  overflow-y: hidden;
  
  /* ✅ 固定定位，防止页面滚动 */
  position: fixed;
  top: 0;
  left: 0;
  right: 0;
  bottom: 0;
  
  z-index: 1;
}

.scalebox-content {
  position: relative;
  overflow: hidden;  /* ✅ 内层再次防溢出 */
}
```

#### 优化价值
- ✅ **100vw/100vh 铺满屏幕**：确保全屏覆盖
- ✅ **三重 overflow 保护**：防止任何方向的溢出
- ✅ **fixed 定位**：彻底杜绝浏览器原生滚动条
