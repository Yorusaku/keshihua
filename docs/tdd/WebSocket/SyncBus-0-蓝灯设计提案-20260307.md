## 🏗️ 一、跨端通信总线设计

### 1.1 文件路径

### 1.2 核心设计说明

**设计原则**：
- ✅ **单例模式**：整个应用仅存在一个 `BroadcastChannel` 实例
- ✅ **类型安全**：使用 TypeScript 接口约束消息格式
- ✅ **生命周期管理**：提供 `unsubscribe` 函数，组件卸载时自动清理

### 1.3 导出接口定义

**文件路径**：`packages/shared/src/websocket/AgvSyncBus.ts`

```typescript
/**
 * 📌 AgvSyncBus 接口定义
 */
export interface AgvSyncBus {
  /**
   * 广播新车数据
   * @param agv 新增的 AGV 数据
   * @description 广播到所有监听 'agv-sync-channel' 的页面（Admin + Dashboard）
   */
  broadcastNewAgv: (agv: IAgvData) => void;

  /**
   * 订阅新车数据
   * @param callback 收到新车数据时的回调函数
   * @returns unsubscribe 函数，用于取消订阅（组件卸载时调用）
   */
  subscribeNewAgv: (callback: (agv: IAgvData) => void) => () => void;
}
```

**文件路径**：`packages/shared/src/websocket/AgvSyncBus.ts`

### 1.5 实现细节

```typescript
/**
 * 📌 BroadcastChannel 实例（私有，避免外部直接访问）
 */
const broadcastChannel = new BroadcastChannel('agv-sync-channel');

/**
 * 📌 监听器集合（用于管理多个订阅者）
 */
const listeners: Set<(agv: IAgvData) => void> = new Set();

/**
 * 📌 消息处理器
 */
const handleMessage = (event: MessageEvent<IAgvData>) => {
  // ✅ 遍历所有监听器，通知新车数据
  listeners.forEach((callback) => {
    callback(event.data);
  });
};

/**
 * 📌 初始化消息监听
 */
const initListener = () => {
  broadcastChannel.addEventListener('message', handleMessage);
};

/**
 * 📌 销毁监听（组件卸载时调用）
 */
const destroyListener = () => {
  broadcastChannel.removeEventListener('message', handleMessage);
};
```

### 1.6 BroadcastChannel 字符串常量

**建议**：将频道名称定义为常量，避免硬编码

```typescript
/**
 * 📌 广播频道名称（常量）

 * - Dashboard 侧：监听 'message' 事件
 */
export const AGV_SYNC_CHANNEL = 'agv-sync-channel';
```

```typescript
export async function addAgv(payload: IAddAgvPayload): Promise<IAgvData> {
  // 500ms 延迟
  await new Promise(resolve => setTimeout(resolve, 500));

  // 组装数据
  const newAgv: IAgvData = {

    x: payload.x,

    status: payload.status,

  return newAgv;
}
```

```typescript
/**
```

```typescript
 * 📌 onUnmounted 钩子：取消订阅，清理资源

  // ✅ 调用 unsubscribe 清理监听

    unsubscribeRef.value();
    unsubscribeRef.value = null;
  }
});
```

### 7.2 监听器生命周期管理

| 场景 | 处理方式 |
|------|----------|
| ✅ 组件挂载 | 订阅消息 |
| ✅ 组件卸载 | 取消订阅（防止内存泄漏） |
| ✅ 多个组件监听 | 使用 `Set` 集合管理多个监听器 |

### 7.3 错误处理

```typescript
/**
 * 📌 错误处理（可选）
 */
const handleMessage = (event: MessageEvent<IAgvData>) => {
  try {
    listeners.forEach((callback) => {
      callback(event.data);
    });
  } catch (error) {
    console.error('AgvSyncBus 消息处理失败:', error);
  }
};
```
