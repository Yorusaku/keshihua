## 1. 需求核心目标与边界

### 1.1 核心问题定义

| 问题类别 | 描述 | 影响范围 |
|---------|------|---------|
| **项目初始化混乱** | 新项目缺乏统一的 monorepo 初始化规范 | 开发效率低，配置不一致 |
| **技术栈落地困难** | Vue3 + Turborepo + Vite 组合缺乏最佳实践 | 技术选型难以落地 |

### 2.2 大屏自适应工作流程

#### 2.2.1 核心数据流

```
┌─────────────────────────────────────────────────────────────┐
│                    大屏自适应数据流                          │
└─────────────────────────────────────────────────────────────┘

┌──────────────┐      ┌──────────────┐      ┌──────────────┐
│  浏览器窗口   │      │ Container    │      │  内容区域    │
│  尺寸变化     │─────▶│  计算 scale  │─────▶│  应用 transform│
│  (resize)    │      │  (Math.min)  │      │  (scale)     │
└──────────────┘      └──────────────┘      └──────────────┘
```

#### 2.2.2 详细步骤说明

| 步骤 | 触发条件 | 处理逻辑 | 输出 |
|------|---------|---------|------|
| **1. 监听 resize** | 窗口尺寸变化 | `window.addEventListener('resize', handleResize)` | 事件触发 |
| **2. 获取窗口尺寸** | resize 事件 | `window.innerWidth`, `window.innerHeight` | 宽高数值 |
| **3. 计算缩放比例** | 获取尺寸后 | `scale = Math.min(width/1920, height/1080)` | 缩放比例 |
| **4. 应用缩放** | 计算完成后 | `transform: scale(${scale})` | 容器视觉变化 |

#### 2.2.3 状态管理

```typescript
// 响应式引用
const screenRef = ref<HTMLElement | null>(null)

// 缩放比例计算（响应式）
const scale = computed(() => {
  const width = window.innerWidth
  const height = window.innerHeight
  
  const widthScale = width / BASE_WIDTH   // 1920
  const heightScale = height / BASE_HEIGHT // 1080
  
  // 取较小值以保持比例，避免内容被裁剪
  return Math.min(widthScale, heightScale)
})

// 应用样式
const containerStyle = computed(() => ({
  transform: `scale(${scale.value})`,
  transformOrigin: 'center center',
}))
```

## 3. 商业级交互与异常场景补全

### 3.1 窗口尺寸异常兜底方案

#### 3.1.1 最小窗口尺寸限制

| 场景 | 窗口尺寸 | 缩放策略 | 用户体验 |
|------|---------|---------|----------|
| **超小窗口** | < 960x540 | 固定缩放 0.5 | 显示内容受限，出现滚动条 |
| **小窗口** | 960x540 ~ 1440x810 | 动态缩放 0.5 ~ 0.75 | 内容缩小，保持比例 |
| **正常窗口** | 1440x810 ~ 2560x1440 | 动态缩放 0.75 ~ 1.0 | 完整显示，比例适配 |
| **大窗口** | > 2560x1440 | 固定缩放 1.0 | 原始尺寸，保持清晰 |

**实现策略**：
```typescript
// 缩放比例限制
const MAX_SCALE = 1.0
const MIN_SCALE = 0.5

const scale = computed(() => {
  const widthScale = window.innerWidth / BASE_WIDTH
  const heightScale = window.innerHeight / BASE_HEIGHT
  const baseScale = Math.min(widthScale, heightScale)
  
  // 限制缩放范围
  return Math.max(MIN_SCALE, Math.min(MAX_SCALE, baseScale))
})
```

#### 3.1.2 安全区域保护

```less
// variables.less
@screen-min-width: 960px;   // 最小安全宽度
@screen-min-height: 540px;  // 最小安全高度
```

### 3.3 性能优化策略

#### 3.3.1 代码分割策略

| 包名 | chunk 分组 | 优化理由 |
|------|-----------|---------|
| `vue`, `vue-router`, `pinia` | `vendor` | 核心框架，更新频率低 |
| `echarts` | `echarts` | 体积大（~600KB），按需引入 |
| `ant-design-vue` | `ant-design-vue` | UI 组件库，组件多 |
| `@antv/s2` | `s2` | 多维分析库，按需引入 |

```typescript
// vite.config.base.ts
export default defineConfig({
  build: {
    rollupOptions: {
      output: {
        manualChunks: {
          vendor: ['vue', 'vue-router', 'pinia'],
          echarts: ['echarts'],
          'ant-design-vue': ['ant-design-vue'],
          s2: ['@antv/s2'],
        },
      },
    },
  },
})
```

#### 3.3.2 按需引入策略

**ECharts 按需引入**：
```typescript
// ❌ 不推荐：全量引入（~600KB）
import * as echarts from 'echarts'

// ✅ 推荐：按需引入（~150KB）
import * as echarts from 'echarts/core'
import { BarChart } from 'echarts/charts'
import { CanvasRenderer } from 'echarts/renderers'
import { TooltipComponent, LegendComponent } from 'echarts/components'

echarts.use([BarChart, CanvasRenderer, TooltipComponent, LegendComponent])
```

#### 3.3.3 防抖节流策略

**Resize 事件防抖**：
```typescript
// ✅ 推荐：使用 lodash-es 防抖
import { debounce } from 'lodash-es'

const handleResize = debounce(() => {
  // 缩放逻辑

```

**API 请求节流**：
```typescript

import { throttle } from 'lodash-es'

const fetchData = throttle(async () => {
  // API 请求逻辑
}, 1000) // 1 秒内最多请求一次
```

### 4.5 大屏自适应技术方案

#### 4.5.1 技术选型对比

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **scale 缩放** | 简单直接，视觉一致 | 像素密度降低 | 大屏展示（✅ 选用） |
| responsive 布局 | 完全自适应 | 需要大量媒体查询 | 响应式网站 |
| vw/vh 单位 | 轻量，无需 JS | 移动端兼容性差 | 移动端适配 |
| flex/grid | 灵活布局 | 复杂度高 | 通用布局 |

#### 4.5.2 实现方案

```vue
<!-- apps/web/src/components/Screen/Container.vue -->
<template>
  <div class="screen-container" :style="containerStyle">
    <div class="screen-content" ref="screenRef">
      <slot />
    </div>
  </div>
</template>

<script setup lang="ts">
import { ref, onMounted, onBeforeUnmount, computed } from 'vue'

// 基准分辨率
const BASE_WIDTH = 1920
const BASE_HEIGHT = 1080

// 缩放限制
const MAX_SCALE = 1.0
const MIN_SCALE = 0.5

// 响应式引用
const screenRef = ref<HTMLElement | null>(null)

// 缩放比例计算
const scale = computed(() => {
  const width = window.innerWidth
  const height = window.innerHeight
  
  const widthScale = width / BASE_WIDTH
  const heightScale = height / BASE_HEIGHT
  
  // 取较小值以保持比例，避免内容被裁剪
  let baseScale = Math.min(widthScale, heightScale)
  
  // 限制缩放范围
  baseScale = Math.max(MIN_SCALE, Math.min(MAX_SCALE, baseScale))
  
  return baseScale
})

// 应用样式
const containerStyle = computed(() => ({
  width: `${BASE_WIDTH}px`,
  height: `${BASE_HEIGHT}px`,
  transform: `scale(${scale.value})`,
  transformOrigin: 'center center',
}))

/**
 * 适配窗口大小变化
 */
const handleResize = () => {
  if (screenRef.value) {
    screenRef.value.style.transform = `scale(${scale.value})`
    screenRef.value.style.transformOrigin = 'center center'
  }
}

// 监听窗口大小变化
onMounted(() => {
  window.addEventListener('resize', handleResize)
})

// 清理事件监听
onBeforeUnmount(() => {
  window.removeEventListener('resize', handleResize)
})
</script>

<style lang="less" scoped>
.screen-container {
  width: 100vw;
  height: 100vh;
  overflow: hidden;
  display: flex;
  justify-content: center;
  align-items: center;
  background-color: @bg-color;
}
</style>
```
