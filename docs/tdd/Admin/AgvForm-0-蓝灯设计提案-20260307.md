```typescript
/**
 * 📌 AGV 新增 Mock 接口（绿灯实现）
 * @description 模拟新增 AGV 数据的 API 调用
 * @param payload 新增数据入参 IAddAgvPayload
 * @returns 新增成功的 AGV 数据
 */
export async function addAgv(payload: IAddAgvPayload): Promise<IAgvData> {
  // ✅ 模拟 500ms 网络延迟（与列表查询保持一致）
  await new Promise(resolve => setTimeout(resolve, 500));

  // ✅ 生成新的 AGV 数据对象
  const newAgv: IAgvData = {
    id: payload.id,
    x: payload.x,
    y: payload.y,
    status: payload.status,
    timestamp: Date.now(),  // ✅ 附加时间戳
  };

  // ✅ 将新数据 push 到 mockAgvData 数组头部（模拟真实新增）
  mockAgvData.unshift(newAgv);

  // ✅ 返回新增的数据（包含生成的 timestamp）
  return newAgv;
}
```

**核心逻辑说明**：
1. **延迟模拟**：使用 `setTimeout` 实现 500ms 网络延迟，与 `fetchAgvList` 保持一致
2. **数据组装**：将 `payload` 中的参数与 `Date.now()` 组装成完整 `IAgvData`
3. **缓存更新**：使用 `unshift()` 将新数据插入到数组**头部**，模拟真实新增场景
4. **返回值**：返回新增的完整数据对象（包含自动生成的 `timestamp`）

**⚠️ 注意事项**：
- ✅ 本次重构不涉及 `mockAgvData` 的持久化，仅在内存中模拟
- ✅ 由于 `mockAgvData` 是模块级常量，每次测试都会重置，避免测试污染

## 🗃️ 二、状态机 Mutation Hook 设计

### 2.1 Hook 文件路径

**文件路径**：`packages/shared/src/network/queries/useAgvMutation.ts`

### 2.2 核心设计说明

**功能需求**：
- ✅ 使用 `@tanstack/vue-query` 的 `useMutation` 包装 `addAgv` 函数
- ✅ 在 `onSuccess` 回调中**自动失效列表缓存**，触发表格静默刷新
- ✅ 支持错误处理和加载状态

### 2.3 核心联动设计

**关键代码**：
```typescript
/**
 * 📌 AGV 新增 Mutation Hook（蓝灯设计）
 */
export function useAddAgvMutation() {
  return useMutation<IAgvData, Error, IAddAgvPayload>({
    // ✅ 使用 addAgv 函数作为 mutation 函数
    mutationFn: addAgv,

    // ✅ 核心联动：成功后自动失效列表缓存
    onSuccess: () => {
      // 使用 queryClient.invalidateQueries 失效 agvList 查询
      queryClient.invalidateQueries({ queryKey: ['agvList'] });
    },
  });
}
```

### 2.4 queryClient 失效机制说明

**失效逻辑**：
```
用户提交表单
    ↓
mutationFn: addAgv() 执行
    ↓
Mock API 更新 mockAgvData（新增数据）
    ↓
onSuccess 回调触发
    ↓
queryClient.invalidateQueries({ queryKey: ['agvList'] })
    ↓
使用 Vue Query 的所有 agvList 查询自动重取
    ↓
AgvList.vue 组件自动静默刷新
    ↓
表格显示最新数据（新增的 AGV 在顶部）
```

### 2.5 返回值类型

**Hook 返回类型**：
```typescript
interface UseAddAgvMutationResult {
  mutate: (variables: IAddAgvPayload) => void;  // 触发新增的函数
  isPending: boolean;                            // 加载状态（Vue Query v5 命名）
  error: Error | null;                           // 错误信息
  isSuccess: boolean;                            // 成功状态
  reset: () => void;                             // 重置状态
}
```

**Vue Query v5 命名差异说明**：
- ❌ 旧版本：`isLoading`、`isError`、`isSuccess`
- ✅ 新版本：`isPending`、`isError`、`isSuccess`

### 3.4 交互流程设计

**新增车辆完整流程**：

```
1. 用户点击"新增车辆"按钮
   ↓
2. 设置 addModalVisible.value = true，弹窗打开
   ↓
3. 用户填写表单（车号、X坐标、Y坐标、状态）
   ↓
4. 用户点击"确认新增"按钮
   ↓
5. 触发 handleFormSubmit → 表单校验
   ↓
6. 校验通过 → mutation.mutate(addForm.value)
   ↓
7. isPending 变为 true，按钮显示加载状态
   ↓
8. Mock API 执行：更新 mockAgvData + 返回新数据
   ↓
9. onSuccess 回调触发 → queryClient.invalidateQueries(['agvList'])
   ↓
10. 列表缓存失效 → AgvList.vue 自动重取数据
   ↓
11. 表格静默刷新，新增的 AGV 显示在顶部
   ↓
12. mutation.isSuccess 变为 true → 关闭弹窗 → 重置表单
```

**关键状态流转**：
| 阶段 | `addModalVisible` | `isPending` | `addForm` |
|------|-------------------|-------------|-----------|
| 初始 | `false` | `false` | 清空 |
| 打开弹窗 | `true` | `false` | 清空 |
| 提交中 | `true` | `true` | 用户输入 |
| 成功 | `false` | `false` | 重置清空 |
| 失败 | `true` | `false` | 用户输入（保留） |
